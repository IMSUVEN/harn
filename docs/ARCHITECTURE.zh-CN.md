# 架构

anima 目前是一个纯文档项目，没有运行时代码。它的「架构」是其文档间的关系以及它所产出的种子。

## 文档层次

```
  哲学（anima 特有）
       |
       | 诠释，不覆盖
       |
  规范 + 指南（学科层面）
```

**Harness 规范**与 **Harness 指南**将 harness 工程表述为学科——一般性理论，取控制视角。**哲学**以培育视角重释同一套公理。这是单向依赖：哲学建基于规范/指南，但规范/指南独立于 anima。

## 种子

```
  哲学 §4（种子应当是什么的理论）
       |
       | 具现为
       |
  seed/（anima init 实际产出的文件）
```

`seed/` 目录包含 `anima init` 将植入用户项目的确切文件。唯一的参数化值是 `seed/AGENTS.md` 中的 `{project-name}`。种子实现了哲学 §4，并通过培育协议回应 §6.2 所揭示的记忆悖论。

## 翻译

`docs/` 中每份文档均有中文翻译（`*.zh-CN.md`）随附。先写英文，再译中文（信达雅标准）。翻译平行维护——英文版的变更须在同一 commit 中对应修改中文版。

## 尚不存在的部分

- **CLI 工具**：`anima init` 命令尚未实现。种子已设计但不可交付。anima 本身的技术栈尚未选定。
- **精灵基础设施**：无 `.anima/` 目录、无持久服务、无生态信号处理。这些属于觉醒阶段（§6.4）。
