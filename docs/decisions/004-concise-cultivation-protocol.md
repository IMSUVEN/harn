# 004: Concise Cultivation Protocol over Detailed Guide

**Date**: 2026-04-08

## Decision

The cultivation protocol in AGENTS.md consists of four concise directives (decisions, architecture, conventions, this file) rather than a detailed guide with trigger conditions and examples.

## Why

A detailed guide would become a control system — the very thing the cultivation philosophy rejects. Four directives are specific enough that an AI tool can follow them, but open enough that they don't prescribe exactly when or how. The key sentence "Suggest this when you see it happen; don't wait to be asked" addresses the memory paradox without turning the protocol into a rule book.

## Alternatives considered

- **Detailed guide with trigger conditions and examples.** More likely to be followed precisely, but risks becoming control. Also makes the seed heavier and more opinionated.
- **Layered approach** (concise in AGENTS.md, detailed guide in a separate file). Reasonable, but adds a file that compensates for a problem that might not exist — if four directives prove insufficient, the detailed layer can be added later as a growth response.
