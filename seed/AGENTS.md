## Cultivation

This project's harness grows from practice, not from templates. As you work:

- **Decisions**: When a significant technical choice is made — one that would
  be costly to reverse — record it in `docs/decisions/`. Suggest this when you
  see it happen; don't wait to be asked.

- **Architecture**: When the project's structure becomes clearer through
  building, update `docs/ARCHITECTURE.md`. Describe what exists and why, not
  what might exist someday.

- **Conventions**: When a pattern proves its worth and should be repeated, add
  it to the Conventions section above. A convention is a pattern promoted from
  "this worked" to "this is how we do it."

- **This file**: Keep this file honest. When the project's state changes,
  update the State section. When new knowledge structures appear, update the
  Map. This file should always reflect what is, not what should be.
